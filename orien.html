The ORIEN Guidelines

These guidelines are designed to help use ORIEN’s brand and assets, including brand of logo, content, trademarks, and web of colors, buttons, typography and forms.

These rules will help you communicate the ORIEN brand most effectively.

Logo

The standard version of our logo, which should be used in most instances, is filled with a solid color called “ORIEN orange”. Black and white version should be used in printed, including paper, clothes or display in small scale (i.e., less than ~50px.).

Do not modify or alter the marks or use them in a confusing way.

Trademark

ORIEN’s trademark should be used in instances where our brand requires name recognition, not just the visual recognition of our logo. It should only be rendered in black or white.

logotype

The logotype describes how the logo and the trademark should properly align with each other. The lockup should only be used with the grey logo and black trademark, or the white version for darker backgrounds.

Spacing

Keep at least a ORIEN logo worth of space between the Logotype and other elements.
When using the logo, it's best to leave about 30% whitespace between it and adjacent elements. and reduce the logo by 10% when putting it inside of a circle.

Usage

  • Don’t add additional shadows, change colors, add lighting effects, Crop, stretch, modify, or change the orientation.
  • Don’t use the logo in confusing or conceptual ways.
  • The "ORIEN" is always capitalized.
    • ORIEN [V]
    • Orien [X]
    • orien [X]
    • oRIEN [x]

Colors

Primary palette

The primary palette includes colors used in the main interface chrome, interaction elements and states. Header, buttons, backgrounds,forms, UI controls, etc.

General usage

  • Use Gray20 for paragraph text
  • Use ORIEN orange for h1 headlines
  • Use ORIEN orange, or Light pink for h2 headlines
  • On the web, use Gray20 as the primary foundational canvas color. All other colors can be used as section or container background fills to break up the canvas or highlight content.

Secondary palette

The secondary palette is used to accent the primary palette, includes colors in content areas and some interaction elements and states.

General usage

  • For text that's intended to be subtle, use Light Gray.
  • Use Cyan, Teal, Pink-red, and Light green for containers.
  • Secondary colors can be used as a section/container background fill only if they're within the context of a higher usage of primary color fills.

Forms

Forms allow users to enter data that is used by the application, or to configure options. Designing forms well requires decisions about its structure and sequence, as well as its interface elements, field labels and offered help and feedback.

Form details

  1. Normal with field label

    When the user engages with the text input field, the floating inline labels move to float above the field.

  2. Normal with input text

    Touching a text field places the cursor and displays the keyboard.

  3. Normal with helper text

    Helper text should appear below the text field and either be persistently visible, or only visible on focus.

  4. Normal with multi-line input text

    Multi-line text fields automatically break to a new line for overflow text and scroll vertically when the cursor reaches the lower edge.

  5. Focus with input text

    When the user engages with the text input field, the floating inline labels move to float above the field.

  6. Focus with multi-input text

    Multi-line text fields automatically break to a new line for overflow text and scroll vertically when the cursor reaches the lower edge.

  7. Error

    When a form or field submission fails, all fields responsible for the problem are shown with a red outline and an exclamation mark icon.
